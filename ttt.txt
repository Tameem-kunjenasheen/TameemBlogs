Business Requirement Document (BRD)

Project Name

NII (Net Interest Income) Calculation Automation

⸻

1. Project Overview

The purpose of this project is to automate the manual process of calculating Net Interest Income (NII) using the Average Balance Report received from the Fee Billing team (MSSOPS).

Currently, the Finance team performs several manual steps in Microsoft Excel, including filtering data, performing calculations, validating balances, and preparing summary reports.

The proposed solution is to develop a Java standalone desktop application that will automatically process the input Excel file, apply business rules, perform required calculations, and generate the final NII report.

Automation will improve accuracy, reduce manual effort, and significantly reduce processing time.

⸻

2. Current Process (As-Is Process)

Step 1 – Receive Average Balance Report

The Finance team receives the Average Balance Report from the Fee Billing team (MSSOPS).

The report contains customer balances for the month.

⸻

Step 2 – Rename and Save File

The file is renamed in the following format:

Avg Bal MMYYYY

Example:

Avg Bal 012024

The file is saved in the shared folder:

Common\Management\Finance\NII Calculation\Calculation

⸻

Step 3 – Create Working Sheet

The source sheet is copied into a new sheet called:

Working

The Working sheet will be used for all NII calculations.

⸻

Step 4 – Filter Currency

Sort the data by Currency (Column D).

Delete all rows where currency is not SAR.

Currencies removed include:

AED
AUD
BHD
EUR
GBP
JPY
KWD
OMR
QAR
USD

Only SAR currency records remain.

⸻

Step 5 – Filter Product Type

Sort the sheet by PRODTYP (Column I).

Delete all product types except:

CUT

Example of removed product types:

HIS
SCA
Other product categories

⸻

3. Core Calculation Logic

Step 6 – Average Balance Calculation (AVC)

A new column named AVC is inserted before column M1.

AVC represents the daily average balance.

Formula used:

AVC = M1 / Number of days in the month

Example:

January → M1 / 31
February → M1 / 28 or 29
September → M1 / 30

The formula is applied for all rows.

Purpose:

M1 contains the total balance for the month.
Dividing by the number of days converts it into the daily average balance.

⸻

Step 7 – Remove Zero Balance Rows

Sort by AVC.

Delete rows where:

AVC = 0

These rows do not contribute to NII calculation.

⸻

4. Customer Classification

Step 8 – Global Customer Mapping

Column A contains GlobalCUST.

Blank values must be filled.

Mapping rules:

If GlobalCUST is blank → set value to “Internal”

If customer belongs to Trust category → set value to “ZZ00”

Special case:

If CUSTID = 086003381
GlobalCUST must be set to “Internal”.

⸻

Step 9 – Unit Classification

Insert a new column before Restriction column (Column H).

Column name:

UNIT

Classification rules:

If GlobalCUST begins with “ZZ” → UNIT = Trust

If GlobalCUST belongs to:

GC
FS
LMN
HHK90-92
SFC
RC
TFS
JADWA

Then UNIT = GC

All other values → UNIT = DCC

⸻

5. Validation Process

Step 10 – HUB Validation

Random validation is performed using HUB screen M971.

Validation fields include:

Customer Number
Product Type = CUT
Currency = SAR
Month Stat = N

At least 4–5 records are randomly checked.

The AVC calculated in Excel must match the balance shown in HUB.

⸻

6. NII Calculation

Step 11 – Interest Rate Application

Calculation template from the previous month is reused.

For each product type, the following formula is used:

NII = AVC × Interest Rate

Interest rate used:

SAMA Reverse Repo Rate

If the interest rate changes during the month, the effective date must be confirmed with SAB.

Example Calculation

Customer AVC = 1,000,000 SAR

Interest Rate = 5%

NII Calculation:

NII = 1,000,000 × 5%

NII = 50,000 SAR annually

Monthly NII:

50,000 / 12 = 4,166.67 SAR

⸻

7. Business Segmentation

Step 12 – Separate Business Reports

NII calculation sheets are prepared separately for:

DCC
GC
Others
Internal

The summary sheet from the previous month is copied and refreshed with new data.

⸻

8. Data Cleanup

Step 13 – Remove Unused Columns

Columns not required for reporting are removed.

Calculation formulas from the previous month template are copied.

Update the following values:

Number of days in month
Interest rate

⸻

9. Final Report Preparation

Step 14 – Sorting

Sort the dataset using:

Client Group
Total NII (USD)

⸻

Step 15 – Summary Table

The summary sheet must contain:

Total AVC
Total NII
Business segment totals

⸻

10. Maker-Checker Control

The final report follows the maker-checker process.

Maker prepares the report.

Checker validates:

Calculations
Data accuracy
Interest rate correctness

⸻

11. Final Action

Once validated:

Finance requests the Fee Billing team to:

Reverse the estimated entries
Book the actual NII values.

⸻

12. Other Reporting Requirements

Regional SSV COO receives the following information via email:

Overall average balance
Computed VOF
NII in USD

⸻

DCC Inputs

The NII report is shared with the Business Finance Operations team.

⸻

Local FCD Reporting

The summary sheet is shared with Local FCD for HORIS inputs.

⸻

SAB Notification

Each month SAB is notified about:

Cash balance
NII values

On a quarterly basis SAB transfers the NII to HBSA as per the PLA.

Iterate through rows:

for (Row row : sheet)

Read required columns:

Currency
ProductType
M1 balance
Customer ID

⸻

Step 3 – Currency Filtering

String currency = row.getCell(currencyColumn).getStringCellValue();

if (!currency.equals(“SAR”))
skip row

⸻

Step 4 – Product Filtering

String productType = row.getCell(productColumn).getStringCellValue();

if (!productType.equals(“CUT”))
skip row

⸻

Step 5 – Calculate AVC

double monthlyBalance = row.getCell(m1Column).getNumericCellValue();

int daysInMonth = 30;

double avc = monthlyBalance / daysInMonth;

⸻

Step 6 – Remove Zero Balance

if (avc == 0)
skip row

⸻

Step 7 – Apply Interest Rate

double interestRate = 0.05;

double nii = avc * interestRate;

⸻

Step 8 – Write Results to Output Excel

Row outputRow = outputSheet.createRow(rowIndex);

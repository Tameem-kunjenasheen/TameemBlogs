Subject: Root Cause Analysis – Monthly Refund Commission File Upload Issue

Dear Imran, Sara, Feras,

Based on our investigation, we found that there is no technical issue with the HUB upload process or the OMS application. The application is working as designed.

Process Overview

The Monthly Refund Commission file uses a sequence number to maintain the upload order. During each upload, the HUB validates the sequence number.

* If the uploaded file contains the expected sequence number, the file is accepted and processed successfully.
* If the sequence number does not match the expected sequence, the HUB rejects the file, and the upload fails.

Therefore, the sequence number must always follow the expected order without any gaps.

Root Cause Analysis

The last successful upload in the HUB was completed in December 2025 (evidence attached).

The January 2026 monthly file was not uploaded. Since no file was processed in January, the expected sequence number in the HUB remained at 811 and did not increment.

However, the subsequent monthly files were uploaded using the next sequence numbers instead of the sequence number that the HUB was still expecting. As a result, the HUB rejected these uploads because of the sequence number mismatch.

If the January 2026 upload issue had been identified and reported at that time, the sequence could have been corrected immediately. However, the issue was not flagged by the Operations team, and the same incorrect sequence number continued to be used in the following monthly uploads.

Findings

* The last successful HUB upload was completed in December 2025.
* The January 2026 upload was not performed.
* Since January was missed, the expected sequence number in the HUB remained 811.
* The subsequent monthly uploads were performed using incorrect sequence numbers, resulting in upload failures.
* This issue continued for approximately six months because the sequence mismatch was not identified.
* The HUB validation worked as designed by rejecting files with an unexpected sequence number.
* There is no technical issue with the HUB upload process or the OMS application.

Conclusion

Based on our investigation, the issue is not caused by an application or technical defect. The root cause is a process/control gap, where the January 2026 upload was missed and the sequence number mismatch was not identified before subsequent monthly uploads were performed.

Please find the attached evidence showing that the last successful upload was completed in December 2025.

Kindly review the Operations process and investigate why the January 2026 upload was missed and why the sequence number mismatch was not identified in the subsequent months.

Please let us know if you require any additional information from the IT team.

Regards,

Tameem Kunjanshasheen

 the screenshot, your colleague is asking you to prepare for these two questions:

1. Can you check how long this process has been in place?
2. Why do you think this control to check the file sits with GM Ops?

These are governance and control questions. Here’s how you can answer them as the IT representative.

⸻

Question 1: How long has this process been in place?

If you know the answer:

“Based on our records, this process has been in place since . I will confirm the exact timeline after reviewing the documentation.”

If you don’t know (which is okay):

“I don’t have the exact date at the moment. I’ll check the historical documentation and confirm when this process was introduced.”

Don’t guess. Compliance teams prefer factual answers.

⸻

Question 2: Why does this control sit with GM Ops?

A good IT answer is:

“My understanding is that GM Ops owns the operational execution of the monthly commission refund process. They upload the file into HUB, review the upload report, and verify that the refunds are processed successfully. Since these are operational activities, the reconciliation and verification control currently sits with the Operations team.”

If they ask why IT doesn’t perform this check:

“IT provides and supports the OMS application and resolves technical issues. The business verification that all customers received the correct refunds is an operational control, which is normally performed by the Operations team.”

⸻

They may ask: Should IT own this control?

Answer:

“From an IT perspective, we can certainly enhance the system by adding automated reconciliation or exception reporting. However, ownership of the operational control is a business decision and should be agreed between the business, Operations, Risk, and Compliance teams.”

⸻

They may ask: Why wasn’t this detected automatically?

Answer:

“Currently, OMS does not have an automated reconciliation or alert mechanism to compare the expected transactions with the uploaded transactions in HUB. The existing process relies on operational review after the upload.”

⸻

They may ask: Can IT build a control?

Answer:

“Yes, we can assess possible enhancements such as:

* Transaction count reconciliation.
* Missing transaction exception reports.
* Upload success/failure alerts.
* Daily reconciliation between OMS and HUB.

We will need to review the technical feasibility and business requirements before implementation.”

⸻

If Compliance asks: Is this a system issue or a control issue?

A safe answer is:

“At this stage, it appears to be a control gap rather than a confirmed system defect. The investigation is still ongoing, and we’ll determine the root cause after completing our analysis.”

⸻

Questions you can ask in the meeting

These questions show you’re thinking about strengthening the process:

* Has this upload and verification process always been handled by GM Ops?
* Is there a documented Standard Operating Procedure (SOP) for this process?
* How often is the HUB report reviewed?
* Is there any reconciliation between OMS transactions and HUB uploads?
* Has this issue occurred before?
* Would the business support implementing an automated reconciliation control in OMS?

⸻

One important tip

During the meeting, avoid statements like:

* “OMS failed.”
* “IT missed it.”
* “The Operations team made a mistake.”

Instead, use neutral language such as:

“We’re investigating the root cause. At this stage, we’ve identified that the current process relies on manual operational verification, and there is an opportunity to strengthen the process by introducing automated reconciliation controls.”

This keeps the discussion objective and professional while avoiding assigning blame before the investigation is complete.”

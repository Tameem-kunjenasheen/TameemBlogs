CR Title

Fix for Arabic Alert Generation Failure Due to Buffer Size Limitation

⸻

Short Description

Increase PL/SQL variable size in ECMDATA.ALERTS package to handle large Arabic alert messages and prevent alert generation failure.

⸻

Description

During execution of ECMDATA.ALERTS.GENERATE_IAI_ALERTS, the process failed while generating KYC alert notifications.

The below Oracle error was observed:

ORA-06502: PL/SQL: numeric or value error:
character string buffer too small

Root cause analysis identified that Arabic alert message content stored in:

P_ALERT_DEFN_REC.ALERT_ENTITY_AR_TEXT

is exceeding the allocated PL/SQL variable size:

V_SMS_TEXT VARCHAR2(4000);

The failure occurs during assignment:

V_SMS_TEXT := P_ALERT_DEFN_REC.ALERT_ENTITY_AR_TEXT;

The Arabic alert content length is larger than the current buffer size, causing the alert generation process to fail.

To fix this issue, the PL/SQL variable size in package body ECMDATA.ALERTS procedure INS_SMS_ALERT will be increased.

Current Variables

V_SMS_TEXT   VARCHAR2(4000);
V_EMAIL_TEXT VARCHAR2(4000);

Proposed Change

V_SMS_TEXT   VARCHAR2(32767);
V_EMAIL_TEXT VARCHAR2(32767);

Additionally, database table column sizes for SMS and Email message fields will be verified.

If required, controlled truncation using SUBSTR() will be implemented before insert operation or the table column size will be increased based on business requirement.

Example:

SUBSTR(message_text, 1, 4000)

⸻

Justification

This change is required to prevent failure in Arabic alert notification generation.

Currently, large Arabic alert messages are exceeding the PL/SQL variable size limit, causing the alert generation process to fail with ORA-06502 error.

This impacts KYC alert processing, SMS notifications, and downstream alert handling.

The proposed fix will ensure larger Arabic Unicode alert messages are processed successfully without buffer overflow issues.

⸻

Implementation Plan

1. Review existing package body ECMDATA.ALERTS procedure INS_SMS_ALERT – DB Team
2. Increase variable size for V_SMS_TEXT and V_EMAIL_TEXT from 4000 to 32767 – DB Team
3. Verify target table column sizes for SMS and Email message fields – DB Team
4. Implement controlled truncation using SUBSTR() if required – DB Team
5. Compile package after modification – DB Team
6. Execute validation testing for Arabic alert generation – Application/DB Team
7. Verify SMS and Email alert generation completion – Application Team

⸻

Business Impact

If this change is not implemented:

* Arabic alert notifications will continue to fail
* KYC alert generation process will not complete successfully
* Business users may miss important customer alerts
* Operational monitoring and compliance-related notifications may be impacted

After implementation:

* Arabic alert messages with larger content will be processed successfully
* Alert generation process will complete without errors
* SMS and email notifications will function normally

⸻

Technical Impact

Current implementation limits alert message variables to 4000 characters, which is insufficient for large Arabic Unicode content.

If this issue is not fixed:

* ORA-06502 buffer overflow errors will continue
* Alert generation jobs will fail intermittently
* SMS and email alert processing will remain unstable
* Downstream notification processing will continue to fail

The proposed change increases PL/SQL variable capacity to support larger message content.

No functional business logic change is involved apart from increasing message handling capacity.

 * =========================================================
 * VoucherBookingService.java
 * =========================================================
 *
 * Voucher Booking Table Calculation Service
 *
 * Excel Mapping:
 *
 * ---------------------------------------------------------
 * ROW   DESCRIPTION
 * ---------------------------------------------------------
 * 904   DCC
 * 905   GC
 * 906   Total
 * 907   VAT
 * ---------------------------------------------------------
 *
 * FORMULAS FROM EXCEL
 * =========================================================
 *
 * B904 = AB875
 * B905 = AB876
 * B906 = B904 + B905
 * B907 = B906 - L863
 *
 * C904 = B904
 * C905 = B905 - E877
 * C906 = SUM(C904:C905)
 * C907 = C906 - G885
 *
 * E904 = C904 / C906 * G893
 * E905 = C905 / C906 * G893
 * E906 = SUM(E904:E905)
 * E907 = E906 - G893
 *
 * F904 = E904 * 0.88
 * F905 = E905 * 0.88
 * F906 = SUM(F904:F905)
 * F907 = F906 - G895
 *
 * G904 = F904 * 0.15
 * G905 = F905 * 0.15
 * G906 = SUM(G904:G905)
 * G907 = G906 - G896
 *
 * =========================================================
 */
public class VoucherBookingService {
    // =====================================================
    // CONSTANTS
    // =====================================================
    public static final double NII_FACTOR = 0.88;
    public static final double VAT_RATE   = 0.15;
    // =====================================================
    // RESULT CLASS
    // =====================================================
    public static class VoucherBookingResult {
        // -------------------------------------------------
        // INPUT VALUES
        // -------------------------------------------------
        public double ab875;
        public double ab876;
        public double l863;
        public double e877;
        public double g885;
        public double g893;
        public double g895;
        public double g896;
        // -------------------------------------------------
        // COLUMN B
        // Avg Balance Before Exceptions
        // -------------------------------------------------
        public double b904;
        public double b905;
        public double b906;
        public double b907;
        // -------------------------------------------------
        // COLUMN C
        // Avg Balance After Exceptions
        // -------------------------------------------------
        public double c904;
        public double c905;
        public double c906;
        public double c907;
        // -------------------------------------------------
        // COLUMN E
        // Vatable NII
        // -------------------------------------------------
        public double e904;
        public double e905;
        public double e906;
        public double e907;
        // -------------------------------------------------
        // COLUMN F
        // NII Estimate to be Booked
        // -------------------------------------------------
        public double f904;
        public double f905;
        public double f906;
        public double f907;
        // -------------------------------------------------
        // COLUMN G
        // VAT
        // -------------------------------------------------
        public double g904;
        public double g905;
        public double g906;
        public double g907;
        @Override
        public String toString() {
            return "\n=============================="
                    + "\nVoucher Booking Result"
                    + "\n=============================="
                    + "\n\nB904 : " + b904
                    + "\nB905 : " + b905
                    + "\nB906 : " + b906
                    + "\nB907 : " + b907
                    + "\n\nC904 : " + c904
                    + "\nC905 : " + c905
                    + "\nC906 : " + c906
                    + "\nC907 : " + c907
                    + "\n\nE904 : " + e904
                    + "\nE905 : " + e905
                    + "\nE906 : " + e906
                    + "\nE907 : " + e907
                    + "\n\nF904 : " + f904
                    + "\nF905 : " + f905
                    + "\nF906 : " + f906
                    + "\nF907 : " + f907
                    + "\n\nG904 : " + g904
                    + "\nG905 : " + g905
                    + "\nG906 : " + g906
                    + "\nG907 : " + g907;
        }
    }
    // =====================================================
    // MAIN CALCULATION METHOD
    // =====================================================
    public VoucherBookingResult calculateVoucherBooking(
            double ab875,
            double ab876,
            double l863,
            double e877,
            double g885,
            double g893,
            double g895,
            double g896
    ) {
        VoucherBookingResult r =
                new VoucherBookingResult();
        // =================================================
        // STORE INPUTS
        // =================================================
        r.ab875 = ab875;
        r.ab876 = ab876;
        r.l863 = l863;
        r.e877 = e877;
        r.g885 = g885;
        r.g893 = g893;
        r.g895 = g895;
        r.g896 = g896;
        // =================================================
        // COLUMN B
        // =================================================
        r.b904 = ab875;
        r.b905 = ab876;
        r.b906 = r.b904 + r.b905;
        r.b907 = r.b906 - l863;
        // =================================================
        // COLUMN C
        // =================================================
        r.c904 = r.b904;
        r.c905 = r.b905 - e877;
        r.c906 = r.c904 + r.c905;
        r.c907 = r.c906 - g885;
        // =================================================
        // COLUMN E
        // =================================================
        r.e904 =
                (r.c904 / r.c906) * g893;
        r.e905 =
                (r.c905 / r.c906) * g893;
        r.e906 =
                r.e904 + r.e905;
        r.e907 =
                r.e906 - g893;
        // =================================================
        // COLUMN F
        // =================================================
        r.f904 =
                r.e904 * NII_FACTOR;
        r.f905 =
                r.e905 * NII_FACTOR;
        r.f906 =
                r.f904 + r.f905;
        r.f907 =
                r.f906 - g895;
        // =================================================
        // COLUMN G
        // =================================================
        r.g904 =
                r.f904 * VAT_RATE;
        r.g905 =
                r.f905 * VAT_RATE;
        r.g906 =
                r.g904 + r.g905;
        r.g907 =
                r.g906 - g896;
        return r;
    }
}

- BalanceSheetService.java
- =========================================================
- 
- Pure calculation service for SAR Balance Sheet (Table 2).
- 
- Calculates:
- C = Before Exceptions
- D = Alinma Exemption  (NIL NII)
- E = PIF Exemption     (Remove)
- F = PIF Special Rate  (0.15%pa)
- G = After Exceptions  (SUM of C:F)
-

- Row 888 : =C886*0.07

  /**
  - Calculates all columns of the SAR Balance Sheet.
  - 
  - All rates and day counts are taken from the constants
  - defined at the top of this class (matching Excel exactly).
  - 
  - @param cEligibleAvgBalance   Row 885 Col C  =L863
  - @param cSarBalance           Row 886 Col C  =C885
  - @param dEligibleAvgBalance   Row 885 Col D  =-E877
  - @param dSarBalance           Row 886 Col D  =D885
  - @param eEligibleAvgBalance   Row 885 Col E  (PIF Exemption input)
  - @param eSarBalance           Row 886 Col E  (PIF Exemption input)
  - @param fEligibleAvgBalance   Row 885 Col F  (PIF Special Rate input)
  - @param fSarBalance           Row 886 Col F  (PIF Special Rate input)
  - @return BalanceSheetResult   object with all 50 computed values
    */
    public BalanceSheetResult calculateBalanceSheet(
    
    ```
     // C — Before Exceptions inputs
     double cEligibleAvgBalance,
     double cSarBalance,
    
     // D — Alinma Exemption inputs
     double dEligibleAvgBalance,
     double dSarBalance,
    
     // E — PIF Exemption inputs
     double eEligibleAvgBalance,
     double eSarBalance,
    
     // F — PIF Special Rate inputs
     double fEligibleAvgBalance,

    ```

  ```
   // ---- Input Validation --------------------------------
   if (YEAR_DAYS <= 0) {
       throw new IllegalArgumentException(
               "YEAR_DAYS must be > 0");
   }
   if (PERIOD_DAYS < 0) {
       throw new IllegalArgumentException(
               "PERIOD_DAYS must be >= 0");
   }
  
   // ---- Create result object ----------------------------
   BalanceSheetResult r = new BalanceSheetResult();
  
   // =====================================================
   // C — BEFORE EXCEPTIONS
   // Excel col C, rows 885-897
   // =====================================================
  
   // Row 885 : =L863
   r.cEligibleAvgBalance = cEligibleAvgBalance;
  
   // Row 886 : =C885
   r.cSarBalance = cSarBalance;
  
   // Row 888 : =C886*0.07
   r.cStatReserve =
           r.cSarBalance * STAT_RESERVE_RATE;
  
   // Row 889 : =C886-C888
   r.cEligibleBalNet =
           r.cSarBalance - r.cStatReserve;
  
   // Row 891 : =(C889*3.75%/365*28)
   r.cVof =
           (r.cEligibleBalNet * VOF_RATE
                   / YEAR_DAYS) * PERIOD_DAYS;
  
   // Row 892 : =C885*0.05%/365*28
   r.cDpf =
           (r.cSarBalance * DPF_RATE
                   / YEAR_DAYS) * PERIOD_DAYS;
  
   // Row 893 : =C891-C892
   r.cHssShare =
           r.cVof - r.cDpf;
  
   // Row 895 : =C893*0.88
   r.cTotalNii =
           r.cHssShare * NII_FACTOR;
  
   // Row 896 : =C895*15%
   r.cVatCost =
           r.cTotalNii * VAT_RATE;
  
   // Row 897 : =SUM(C895:C896)
   r.cClaimFromSabb =
           r.cTotalNii + r.cVatCost;
  
   // =====================================================
   // D — ALINMA EXEMPTION (NIL NII)
   // Excel col D, rows 885-897
   // =====================================================
  
   // Row 885 : =-E877
   r.dEligibleAvgBalance = dEligibleAvgBalance;
  
   // Row 886 : =D885
   r.dSarBalance = dSarBalance;
  
   // Row 888 : =D886*0.07
   r.dStatReserve =
           r.dSarBalance * STAT_RESERVE_RATE;
  
   // Row 889 : =D886-D888
   r.dEligibleBalNet =
           r.dSarBalance - r.dStatReserve;
  
   // Row 891 : =(D889*3.75%/365*28)
   r.dVof =
           (r.dEligibleBalNet * VOF_RATE
                   / YEAR_DAYS) * PERIOD_DAYS;
  
   // Row 892 : =D885*0.05%/365*28
   r.dDpf =
           (r.dSarBalance * DPF_RATE
                   / YEAR_DAYS) * PERIOD_DAYS;
  
   // Row 893 : =D891-D892
   r.dHssShare =
           r.dVof - r.dDpf;
  
   // Row 895 : =D893*0.88
   r.dTotalNii =
           r.dHssShare * NII_FACTOR;
  
   // Row 896 : =D895*15%
   r.dVatCost =
           r.dTotalNii * VAT_RATE;
  
   // Row 897 : =SUM(D895:D896)
   r.dClaimFromSabb =
           r.dTotalNii + r.dVatCost;
  
   // =====================================================
   // E — PIF EXEMPTION (Remove)
   // Excel col E, rows 885-897
   // =====================================================
  
   r.eEligibleAvgBalance = eEligibleAvgBalance;
  
   r.eSarBalance = eSarBalance;
  
   r.eStatReserve =
           r.eSarBalance * STAT_RESERVE_RATE;
  
   r.eEligibleBalNet =
           r.eSarBalance - r.eStatReserve;
  
   r.eVof =
           (r.eEligibleBalNet * VOF_RATE

           (r.eSarBalance * DPF_RATE
                   / YEAR_DAYS) * PERIOD_DAYS;
  
   r.eHssShare =
           r.eVof - r.eDpf;
  
   r.eTotalNii =
           r.eHssShare * NII_FACTOR;
  
   r.eVatCost =
           r.eTotalNii * VAT_RATE;
  
   r.eClaimFromSabb =
           r.eTotalNii + r.eVatCost;

   // Excel col F, rows 885-897
   // =====================================================

   r.fSarBalance = fSarBalance;
  
   r.fStatReserve =
           r.fSarBalance * STAT_RESERVE_RATE;
  
   r.fEligibleBalNet =
           r.fSarBalance - r.fStatReserve;
  
   r.fVof =
           (r.fEligibleBalNet * VOF_RATE
                   / YEAR_DAYS) * PERIOD_DAYS;
  
   r.fDpf =
           (r.fSarBalance * DPF_RATE
                   / YEAR_DAYS) * PERIOD_DAYS;
  
   r.fHssShare =
           r.fVof - r.fDpf;
  
   r.fTotalNii =

   r.fVatCost =
  ```

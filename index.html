📄 BUSINESS REQUIREMENT DOCUMENT (BRD)

Project: NII Estimation Automation

⸻

1. 📌 Purpose

The purpose of this application is to automate the NII (Net Interest Income) estimation process, which is currently done manually in Excel.

This will:
	•	Reduce manual work
	•	Avoid errors
	•	Save time
	•	Provide consistent output

⸻

2. 📌 Scope

This application will:

✔ Read input data from Excel files
✔ Filter and process data based on business rules
✔ Calculate balances and category-wise values
✔ Apply formulas (same as Excel)
✔ Generate final NII output

⸻

3. 📌 Input Files

The system will use below input files:

3.1 Source File

Contains raw data with columns:
	•	Bank (ALDCG)
	•	Customer Number (ALDCS)
	•	Product Type (ALPRIE)
	•	Currency (ALCYCD)
	•	Balance (ALMVC1 → in Halalas)

⸻

3.2 Category File

Contains customer classification:
	•	TRUST
	•	DCC
	•	GC
	•	PRIVATE FUND
	•	OTHER / INTERNAL

⸻

3.3 Exception File

Contains special customers:
	•	Alimna Exception
	•	PIF Exception

⸻

3.4 Percentage File

Contains:
	•	VOF %
	•	VAT %
	•	Any other rates

⸻

4. 📌 Data Processing Steps

Step 1: Filter Source Data
	•	Keep only CUT products
	•	Keep only SAR currency
	•	Keep only SABB bank
	•	Remove rows with zero balance

⸻

Step 2: Convert Balance
	•	Column ALMVC1 is in Halalas
	•	Convert to SAR:

Balance = ALMVC1 / 100


⸻

Step 3: Total Balance Calculation
	•	Sum all filtered balances
	•	This is Total Avg Balance

⸻

Step 4: Category Split (5 Tables)

System will create 5 tables:
	1.	TRUST
	2.	DCC
	3.	GC
	4.	PRIVATE FUND
	5.	OTHER / INTERNAL

For each category:
	•	Pick customer list
	•	Match with source data (ALDCS)
	•	Apply SUMIF logic
	•	Calculate balance per customer
	•	Calculate total per category

⸻

Step 5: Exception Calculation

5.1 Alimna Exception
	•	Take customer list from exception file
	•	Calculate SUMIF
	•	Store total

5.2 PIF Exception
	•	Same logic as above

⸻

Step 6: Eligible Balance Calculation

Eligible Balance = Total Balance - Private Fund Balance


⸻

Step 7: DCC and GC Split
	•	DCC includes:
	•	Trust
	•	Internal / Other
	•	GC includes:
	•	Remaining balance

⸻

Step 8: Apply Financial Formulas

8.1 VOF Calculation

VOF = Eligible Balance × Rate × (Days / 360)


⸻

8.2 Deposit Protection Fund

DPF = Eligible Balance × % × (Days / 360)


⸻

8.3 HSS Share

HSS Share = VOF - DPF


⸻

8.4 VAT Calculation

VAT = NII × VAT %


⸻

8.5 Final Claim

Final Claim = NII + VAT


⸻

5. 📌 Output

The system will generate:

✔ Category-wise tables (like Excel)
✔ Total balances
✔ NII calculation
✔ VAT and final claim

## 2. The "5 Whys" for Physical Workstations
Since you've confirmed it's a physical box, we can get more granular with the root cause analysis:
**Problem: Screen lag and intermittent "Connection Lost" pop-ups.**
 1. **Why is the connection dropping?**
   * *Answer:* The Bloomberg bb.exe process is losing its heartbeat signal to the Bloomberg router.
 2. **Why is the heartbeat signal being lost?**
   * *Answer:* The local network interface card (NIC) is dropping packets or entering a "Power Save" state.
 3. **Why is the NIC dropping packets?**
   * *Answer:* There is a physical layer mismatch (bad cable) or the workstation's security software (Firewall/EDR) is deep-packet inspecting Bloomberg's encrypted stream.
 4. **Why is the security software interrupting the stream?**
   * *Answer:* The Bloomberg executable paths and specific ports (8194-8198) haven't been "Whitelisted" or excluded from active scanning.
 5. **Why were they not whitelisted? (Root Cause)**
   * *Answer:* **Incomplete IT Security Policy for Trading Desks.** Professional trading apps require specialized bypasses for low-latency data streams that typical office apps (like Word or Outlook) do not.
## 3. High-Priority Debugging Steps (Local Workstation)
### A. The "DNET" Diagnostic
Type **DNET ** on your terminal immediately.
 * Check the **Latency** and **Packet Loss**.
 * If you see any red text, click **"Run Repair"** or **"Restart All Services."** This resets the local Bloomberg communications engine without rebooting your PC.
### B. Windows Power Settings
On physical workstations, Windows often tries to save power on the Ethernet port.
 1. Go to **Device Manager** > **Network Adapters**.
 2. Right-click your Controller (e.g., Intel Ethernet) > **Properties** > **Power Management**.
 3. **Uncheck** "Allow the computer to turn off this device to save power."
### C. Resource Monitoring
Bloomberg is a "Heavy" application. If you have many **Launchpad** monitors or **Live Excel API** links:
 * Open Task Manager (Ctrl+Shift+Esc).
 * Look for **bb.exe** and **b-pipe**.
 * If CPU usage is consistently above 70%, your workstation hardware may be the bottleneck.
